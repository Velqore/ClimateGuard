# Contributing to ClimateGuard

Thanks for helping improve ClimateGuard. This repository is designed to be runnable from source, verifiable with tests, and easy to review.

## Before you start

- Read the README and verify the project runs locally.
- Run `npm test` from the repository root.
- Run `npm run build` to confirm the frontend builds.

## Development workflow

1. Create a branch for your change.
2. Make the smallest focused change that solves the issue.
3. Add or update tests when behavior changes.
4. Re-run `npm test` and `npm run build` before opening a pull request.

## Style

- Keep changes small and consistent with the existing codebase.
- Prefer clear, direct code over clever abstractions.
- Do not add new dependencies unless they solve a real problem.

## Good first contributions

- Improve README clarity.
- Add small UI polish fixes.
- Add more endpoint checks or lightweight tests.
- Improve accessibility and error handling.

## Pull request checklist

- The app still runs locally.
- The build passes.
- The change is explained clearly.
- Screenshots are included for UI updates.
